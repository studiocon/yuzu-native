## ゴール

両リポの共有ファイルの差分を「意図的差分（RN 対応・プラットフォーム差）」と「取り込み漏れ（yuzu-app 側の修正が native に未反映）」に分類した判定表を出すこと。

## 対象の特定方法

固定リストを直書きせず、`lib/` 配下で yuzu-app の `lib/` に同名ファイルがあるものを対象と定義する。例（あくまで例。実際の対象は都度洗い出す）: `period.ts` / `streak.ts` / `sentimentSeries.ts` / `voiceprint.ts` / `stats.ts` / `prompts.ts`。

## 成功基準

全対象ファイルについて以下を示した表を出す。

- diff の有無
- 分類（意図的差分 / 取り込み漏れ）とその根拠（該当行）
- 取り込み漏れがある場合は修正提案

取り込み漏れがあっても勝手に取り込まない。適用はユーザー判断とする。
