## ゴール

変更・追加した全 UI 文言が `DESIGN.md` §4 に準拠していること。

## 正典への到達方法

`../yuzu-app/DESIGN.md` の §4（VOICE & TONE）を読む。ローカルに無ければ GitHub `studiocon/yuzu-app` の `DESIGN.md` を参照する。

NG ワード表や UI コピー表はこのスキルに複製しない。正典とのドリフトを防ぐため、判定のたびに §4 を読みに行く。

## 成功基準

対象の文言それぞれについて以下をチェックする。

- [ ] **NG ワード 0**: 正典の NG ワード表と照合し、該当語が混入していない
- [ ] **句点ルール準拠**: UI 要素（見出し・ボタン・状態 pill・短い命令）は句点なし。状況を説明する本文（複数文の説明・エラー・トースト等）のみ句点あり
- [ ] **英日使い分け**: 英語＝状態・挑発（Unbounded の pill/バッジ）、日本語＝事実・本文
- [ ] **トーン**: 命令形・断定形（NIKE 寄り）。詩的にしない・励まさない・優しくしない

## 報告

照合した文言と判定結果を対応表で示す。各判定には証拠として正典の該当行（行番号または引用）を添える。
