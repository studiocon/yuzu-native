## ゴール

変更・追加した UI の見た目（色・角丸・余白・box-shadow・コンポーネント構造・アニメーション）が `DESIGN.md` に準拠していること。文言（§4 VOICE & TONE）は対象外（`copy-review` を使う）。

## 正典への到達方法

`DESIGN.md`（このリポジトリのルート）を読む。冒頭の「ネイティブ版での位置づけ」注記に、yuzu-app 側の正典コミットハッシュとネイティブ側の意図的差分一覧がある。

このファイルは yuzu-app からの**移植コピー**であり、yuzu-app 本体が正典（source of truth）。以下の手順で鮮度を確認してから照合する。

1. `git -C ../yuzu-app log -1 --format=%H -- DESIGN.md` で yuzu-app 側の最新コミットハッシュを取得する（ローカルに無ければ GitHub `studiocon/yuzu-app` の `DESIGN.md` を参照）。
2. ネイティブ側冒頭の注記に書かれたハッシュと比較する。**異なる場合はレビュー前にその旨を報告し、yuzu-app 側の差分（コミットログ・diff）も確認した上で続行するか確認する**（古い正典と照合しても意味が無いため）。

NG ワード表・コンポーネント仕様の値はこのスキルに複製しない。判定のたびに `DESIGN.md` を読みに行く。

## 成功基準

対象の変更それぞれについて以下をチェックする。

- [ ] **カラートークン**: 使用色が §2 カラーパレット / 該当コンポーネント節（§6）の値と一致するか。新しい色を増やしていないか
- [ ] **シェイプ**: 角丸が `--radius-0` / `--radius-sharp`(2px) / `--radius-pill` のいずれかか。blob 形状を使っていないか
- [ ] **シャドウ**: `box-shadow` / `drop-shadow` を使っていないか。使っている場合、§5 で許可された3セレクタ（`.tab-bar` / `.fab-record` / `.onboarding-mic` 相当のネイティブコンポーネント）に限定されているか
- [ ] **コンポーネントパターン**: §6 の該当コンポーネント仕様（構造・状態・インタラクション）と一致するか
- [ ] **アニメーション**: §7 の easing 関数・`prefers-reduced-motion` 相当（RN では `AccessibilityInfo.isReduceMotionEnabled` 等）に沿っているか
- [ ] **意図的差分の記録**: 正典と違う実装にした場合、`DESIGN.md` 冒頭の「ネイティブ側の意図的な差分」リストに追記されているか（未記載なら追記を提案する）

## 報告

チェック項目ごとに対応表（項目 / 判定 / 証拠となる `DESIGN.md` の該当行 or 該当コード行）を示す。正典との差分が見つかった場合は「意図的差分（記録が必要）」か「取り込み漏れ・単純な不一致」かを分類する。差分があっても勝手に修正しない。適用はユーザー判断とする。
